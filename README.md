# Pair Version

This folder contains the task manager app built using an AI pair programming assistant.

**Tool used:** GitHub Copilot  
**Time to build:** ~1.5–2 hours  
**Suggestions accepted:** ~15  
**Suggestions rejected:** ~8  
**Live URL:** https://anishagrawal25.github.io/Project-Enginnering-track/

## Notes

The AI assistant helped generate boilerplate code, suggest functions, and speed up development. However, I actively guided the structure and logic, making decisions about how the app should behave.

I accepted suggestions for basic functionality like adding tasks and handling input, but I modified or rejected suggestions when I wanted simpler or more understandable logic.

This approach helped me understand the code deeply, since I was involved in every step. Compared to the vibe version, this was slower but much easier to explain and customize.

# Vibe vs. Pair Challenge

This challenge involves building the same Task Manager application twice to compare two distinct AI-assisted development workflows: **Vibe Coding** and **AI Pair Programming**.

## Comparison Table

| Dimension          | Vibe Version                     | Pair Version             |
| :----------------- | :------------------------------- | :----------------------- |
| **Speed**          | Very fast (generated in minutes) | Slower (manual + guided) |
| **Control**        | Low control                      | High control             |
| **Code Quality**   | Good but complex                 | Simple and readable      |
| **Explainability** | Hard to explain                  | Easy to explain          |
| **Editability**    | Hard to modify                   | Easy to modify           |
